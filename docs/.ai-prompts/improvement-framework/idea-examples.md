## Example 2: CRO Idea

## [CRO-001] Exit-Intent Popup with Email Capture

**Category**: cro
**Type**: experiment
**Priority**: 9/10
**Effort**: s
**Impact**: +15% recovery, +20% email list growth
**Confidence**: high

### Description

- Offer 10% off first order in exchange for email
- Show clean, on-brand design
- Be dismissible with "No thanks, I pay full price"
- Only show once per user (30-day cookie)
- Mobile-friendly (trigger on scroll-up gesture)

### User Value
- Get a discount on first purchase
- Learn about future promotions
- Feel like they're getting insider access

### Business Value
- Recover 15-20% of abandoning visitors
- Grow email list for remarketing
- Gather first-party data (valuable with iOS changes)
- Track effectiveness by traffic source

### Technical Notes
- Use exit-intent detection library (e.g., ExitIntent.js)
- Integrate with Klaviyo or existing email system
- Store dismissal in localStorage or cookie
- A/B test offers: 10% off vs free shipping vs $5 off
- Track conversion in PostHog

### Success Metrics
- Popup display rate: 60% of visitors
- Email capture rate: 15% of displayed popups
- Conversion from captured emails: 8-12%
- Incremental revenue: $5,000/month (at 1,000 captures/month)
- Email list growth: +1,000 subscribers/month

### Dependencies
- Email marketing platform (Klaviyo recommended)
- Discount code creation in Shopify
- Legal compliance (GDPR, CAN-SPAM)

### Risks
- Some users find popups annoying
- Could train users to always wait for popup
- Legal compliance needed
- Mobile experience challenges

### Similar Implementations
- **Mejuri**: Clean, timed popup with discount

## Example 6: Technical Idea

## [TECH-001] Implement Image Lazy Loading

**Category**: technical
**Type**: optimization
**Priority**: 8/10
**Effort**: xs
**Impact**: -30% LCP, +10% mobile conversion
**Confidence**: high

### Description
Implement native lazy loading for all below-the-fold images using the `loading="lazy"` attribute. This will:
- Defer image loading until user scrolls near them
- Reduce initial page load time
- Improve LCP (Largest Contentful Paint)
- Save bandwidth for users who don't scroll

### User Value
- Faster initial page load
- Smoother scrolling experience
- Less data usage on mobile

### Business Value
- Improved Core Web Vitals scores
- Better mobile experience
- Higher conversion on slow connections
- SEO benefit from faster load times

### Technical Notes
- Add `loading="lazy"` to all `<img>` tags below fold
- Keep hero image as eager load
- Add `decoding="async"` for non-critical images
- Add width/height to prevent layout shift
- Test LCP improvement in PageSpeed Insights

### Success Metrics
- LCP improvement: -30% (faster is better)
- Mobile conversion: +8-10%
- Bounce rate: -5%
- Core Web Vitals: "Good" rating

### Dependencies
- Audit all image usage
- Identify above/below fold images
- Test loading behavior

### Risks
- Minimal risk (browser-native feature)
- Ensure critical images aren't lazy
- Test for layout shifts

### Similar Implementations
- Standard practice for modern sites
- Used by all major e-commerce sites

## Quality Checklist for Generated Ideas

Every idea should pass these checks:

✅ **Specific**: Clear what to implement
✅ **Actionable**: Can be built directly
✅ **Measured**: Has success metrics
✅ **Prioritized**: Clear effort/impact
✅ **Contextual**: Fits user and business needs
✅ **Complete**: All sections filled out
✅ **Realistic**: Achievable with current resources
✅ **Valuable**: Clear user and business value

If an idea doesn't meet these criteria, refine it before outputting.
