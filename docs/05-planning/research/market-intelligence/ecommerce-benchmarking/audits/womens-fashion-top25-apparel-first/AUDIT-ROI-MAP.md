Interpretation:
- Higher `signal_score` generally means more “interesting” mechanics worth copying (returns portals, reviews tooling, personalization, etc).
- `blocked_pct` is snapshot-based; treat it as “automation risk” not a definitive bot-protection verdict.
- Cart/checkout behaviors still require screenshots to confirm; use the checkout triage report to pick stores that are most likely to yield strong checkout evidence quickly.
- `signal_score` is **not a quality score** — it is mostly “tooling signal density” from static HTML and can undercount stores that implement features in first-party code or where the scan misses JS-rendered surfaces.
